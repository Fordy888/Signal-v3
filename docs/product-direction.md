# DTL Signal — Product Direction

> Captured 16 June 2026. This is a north star document, not a feature backlog.
> No new features until 7 consecutive clean hosted deliveries are confirmed.

---

## Positioning

Signal is not a newsletter.

Signal is a **Personal AI Information and Learning Assistant**.

Long-term mission: **Improve executive judgment.**

---

## Current Architecture

```
Ingest → Score → Synthesise → Deliver
```

## Future Architecture

```
Subscriber Profile
  → Intelligence Profile
    → Scoring (personalised)
      → Personalisation
        → Learning
          → Delivery
```

The pipeline remains unchanged. The Intelligence Profile becomes the primary object in the system.

---

## Intelligence Profile Framework

Signal onboarding is not a traditional signup form. It is the creation of an **Intelligence Profile**.

Signal should understand:

- Professional context
- Industries operated in
- Industries learned from
- Opportunities being pursued
- Threats being monitored
- Intelligence style preferences

This profile becomes the foundation of all future personalisation.

---

## Product Evolution

Signal evolves through three layers:

| Layer | Question | Value |
|-------|----------|-------|
| **Information** | What happened? | Awareness |
| **Intelligence** | Why does it matter? | Context |
| **Learning** | What should I understand? | Capability |

The long-term opportunity is a daily learning experience that compounds executive capability over time. No courses. No exams. No certificates. Just consistent capability development through daily exposure.

---

## Signal Platinum (Working Concept)

- 10 minutes per day
- Continuous executive development
- AI-powered
- Highly personalised
- Positioned against traditional L&D (which executives avoid) while delivering the same outcome (improved judgment) through a format they already engage with

---

## Strategic Observation

Signal's real asset is not content.

Signal's real assets are:

1. **Intelligence Profiles** — accumulated understanding of each subscriber
2. **Behavioural Learning** — what they engage with, skip, act on
3. **Executive Memory** — compounding context that makes each brief more valuable than the last

This creates the foundation for a highly personalised information and capability development platform.

Signal is evolving from a content product into an intelligence platform.

---

## Immediate Priorities (Unchanged)

No new features until operational reliability is proven.

Success criteria before any product development:

1. Successful hosted deployment ✅
2. Successful manual run ✅
3. Seven consecutive successful morning deliveries ← current milestone

Only then should subscriber scaling commence.

---

## Sequencing

1. Prove reliability (7 clean deliveries)
2. Onboard Founding Members on v1
3. Observe behaviour via Founder Intelligence Dashboard
4. Let Intelligence Profiles and personalisation emerge from real data
5. Build personalisation layer informed by actual subscriber patterns
6. Introduce Signal Platinum when the learning layer has evidence behind it
